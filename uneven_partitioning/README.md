# Uneven Partitioning

## 非均匀切分优化策略

### 预测器设计

经过大量实验测试表明，流水线并行计算过程中设备的逐批次推理时延以及期间的平均功耗，主要与设备所部署的模型层数以及批处理配置的批次大小有关，这其中的数据因设备的异构性而展现出差异化规律，总体呈现出复杂的非线性关系。

故我们提出通过训练小规模的神经网络来对各设备的时延和能耗做精准预测，以捕获复杂的模型层数与批次特征关系并尝试拟合出时延及能耗表现，泛化至未知的模型层数比例及批次大小情况，从而辅助仿真器的模拟仿真，这其中的必要性在于，当确定各设备所部署的模型层数以及理论最大批次大小，即确定模型切分比例后，流水线并行推理仿真器的实现则主要依赖于各设备中每个批次的推理时延，并以此模拟吞吐量表现，同时由于系统以能效比为目标，故还需要所有设备的功耗表现数据，以计算当前模型切分策略下，流水线并行最终的能效比表现。

批时延及功耗预测器的结构设计为一个多层的神经网络，以设备所部署的模型层数以及批大小构成的二维特征作为输入，而网络的输出则是相应时延及功耗表现的二维表示。由于不同设备的表现存在异构性，故需要为流水线中的每一个设备训练一个预测器，训练数据使用预基准测试所收集的相应设备数据。

### 仿真器设计

流水线仿真器的实现依赖于时延及功耗预测器，目标是根据给定预测器所预估的批次时延及功耗结果，在给定的工作负载下，快速模拟出完整的推理过程，得到总处理时延进而得到吞吐量及能耗表现，最终仿真所得的能效比用于启发式模型切分算法迭代优化。

基于底层推理引擎的推理特性，我们对流水线并行推理的特性进行分析，模拟过程需要遵守一定约束，当上线流水线处于计算状态的批次数量达到理论最大值，即流水线设备数量时，必须等待最早批次计算完成，才可以开始下一批次的计算。同时，对单个设备而言，两个连续批次的计算之间至少存在基本固定的数据处理等其他操作时延。此外，在稳定的网络通信环境下，节点间中间结果的传输时延可认为与批次的计算量具有线性关系，而批次计算量同样取决于模型层数以及批次大小。

### 模型切分算法

基于预测器和仿真器，暴力穷举所有可能的模型切分方案相对耗时，有必要优化搜索效率，故本文提出一种能效感知的贪心局部搜索算法，用于在异构设备组成的流水线并行推理系统中高效探索模型非均匀切分策略。

算法从一个可行的初始模型切分方案出发，基于设备算力进行启发式初始化，并在每一轮迭代中枚举所有可能的局部调整操作，即将一个模型层从某一设备迁移至另一设备。在每个候选切分方案下，算法调用设备级批次推理时延与功耗预测器估计各设备的批处理性能，并通过流水线仿真器快速模拟得到系统整体的吞吐量与功耗，从而得到对应的能效比。随后，算法贪心地选择能够带来最大能效提升的层迁移操作并立即执行，将此时的模型切分比例作为当前最优解。该过程不断重复，直至不存在任何能够进一步提升能效比的局部调整为止。

## Pre-Benchmark

`pre_benchmark.py`用于进行流水线的前期基础推理测试，以收集预测器的训练数据。需要指定所使用的模型，以及参与推理测试的设备，在不同的批次大小以及模型层数下进行推理测试，收集平均批次推理时延以及平均功耗表现数据。

开始测试之前，根据会根据所使用的模型，以及各设备所剩余可用的显存空间，决定需要测试的层数范围以及批次大小范围。

实际推理测试中，收集的用于训练预测器的数据格式保存为json格式。其中，`model`表示当前使用的模型（不同的模型需要训练不同的预测器）；`devices`表示参与推理测试的设备名（不同的设备需要训练各自的预测器）；`num_layer`表示各设备都部署了相同的若干层数；`batch_sizes`表示固定当前num_layer层数下所测试的批次大小有哪些；`batch_latencys_ms`表示各设备对应各批次大小下的批次平均推理时间(ms)；`powers_W`表示各设备对应各批次大小下的平均推理功耗(W)。依此类推，接下来的数据就是num_layer+1下各批次大小的测试表现数据。

## Predictor Training

`pipeline_predictor.py`脚本则使用上述收集的训练数据训练预测器，每个设备将训练独立的预测器。完整流程如下：

- 读取你给定格式的 JSON 数据

- 按 `模型名 × 设备名` 自动整理训练样本

- 以二维输入 `[(num_layer), (batch_size)]` 训练二维输出 `[(latency_ms), (power_W)]` 的小型 MLP 预测器

- 为每个设备分别保存权重，命名形式类似 `opt-1.3b-jetson-64g-4.pt`

- 保存时同时把归一化参数一并存入 checkpoint，后续加载后可直接预测

- 支持命令行训练和命令行预测，也支持在 Python 代码里直接调用

其中，每个 `模型-设备` 单独训练一个预测器，符合异构设备设定。训练时自动对输入和输出做标准化，避免时延和功耗量纲不同导致训练不稳。此外，checkpoint 里除了 `state_dict`，还保存了 `x_mean/x_std/y_mean/y_std`，所以后续预测时不会丢失归一化信息。预测器训练完成后，会输出一个 `training_summary.json`，可查看每个设备预测器的误差指标。

直接为所有设备训练：

```Bash
python pipeline_predictor.py train \
  --json_path prebenchmark_data.json \
  --save_dir checkpoints
```

只训练某个模型或某个设备：

```Bash
python pipeline_predictor.py train \
  --json_path your_data.json \
  --save_dir checkpoints \
  --only_model opt-1.3b \
  --only_device jetson-64g-4
```

训练完成后，会在 checkpoints/ 下得到类似这些文件：

```
checkpoints/
  opt-1.3b-jetson-64g-4.pt
  opt-1.3b-jetson-16g-2.pt
  opt-1.3b-jetson-16g-8.pt
  opt-1.3b-jetson-8g-1.pt
  training_summary.json
```

在 Python 里直接加载并预测：

```Python
from pipeline_predictor import load_predictor

predictor = load_predictor("checkpoints/opt-1.3b-jetson-64g-4.pt", device="cuda")
latency_ms, power_w = predictor.predict(num_layer=12, batch_size=64)

print("latency_ms =", latency_ms)
print("power_w    =", power_w)
```
